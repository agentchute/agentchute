The protocol spec

Working draft v1; reference CLI v0.2. The canonical version is AGENTCHUTE.md in the repo — it renders cleanly on GitHub with the right anchor links, table of contents, and code highlighting. The summary below is a reading guide for what's in each section.

The TL;DR

Protocol primitives (medium-agnostic): per-recipient inbox · ordered identified messages · no-overwrite delivery · recipient-owned consumption · recipient-side polling · optional wake · self-registration.

Reference implementation (v0.2): markdown files on a shared filesystem · atomic create + rename · vendor-namespaced dotdir at repo root · three-tier polling model · tmux, HTTP, SSH, and OS notification wake adapters.

Alternate inbox transports (Redis/NATS queues, S3-prefixed inboxes, HTTP endpoints, git-backed) are protocol-compatible — see EXTENSIONS.md.

Section guide

  • §1 Purpose — the inbox primitive vs. the filesystem reference implementation.
  • §2 Scope — what's in v1, the concurrency posture, access requirements.
  • §3 Layout (filesystem reference) — how the CLI maps primitives onto a dotdir.
  • §4 Discovery — control-repo and loop-dir cascades; cross-folder enrollment via pointers.
  • §5 Registration — the canonical registration shape and operational metadata.
  • §6 Messaging — filename format, sender/recipient flows, reply obligations.
  • §7 Coordination defaults — authority rules, pre-authorization carve-outs, pool-scoped identity.
  • §8 Wake adapters (best-effort) — the adapter contract (§8.1), and the **wake responsibility** rule (§8.2): recipients poll; adapters are optional optimizations.
  • §9 Liveness — last_seen semantics and manual-vs-automated health.
  • §10 Watchdog — pool-side monitoring (§10.1) and the **recipient-side watcher** (§10.9) sibling.
  • §11 Protocol correction (best effort) — quarantine actions and the doctor diagnostic tool (§11.6).
  • §12 Non-goals (v1) — things v1 deliberately does not address.
  • §13 v2 deferred items — remote notifications, transcript export, role assignment.
  • §14 Vendor implementations — multiple vendor dotdirs can coexist.
  • Appendix B — Hook templates for Claude Code, Codex, and Gemini.

Reading the spec

Open AGENTCHUTE.md on GitHub for the full text. Grab it locally: 

curl -fsSL https://raw.githubusercontent.com/agentchute/agentchute/main/AGENTCHUTE.md -O

The protocol primitives in §1 are the contract. The filesystem sections (§3–§6) are how the reference CLI maps those primitives to disk.
